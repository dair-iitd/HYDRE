# HYDRE: Retrieval-based In-context Learning for Few-shot DSRE

Implementation of the HYDRE pipeline from https://openreview.net/pdf?id=Jovc64vlDb.

## Quick Start

```bash
source $HOME/.local/bin/env && uv sync
```

## Setup

**Prerequisites:** `uv`, CUDA GPU (Stages 1-3), API key (Stage 4).

**Required files:**
- `PARE/ckpt/bert-base-uncased_512_32_5_1e-5_772_nyt10m_sep_na.pth.tar` — download from https://github.com/dair-iitd/DSRE
- `datasets/nyt10m/eng_Latn_final.jsonl` — input queries
- `datasets/nyt10m/nyt10m_rel2id.json` — relation mapping (generated by download script)
- `nyt10mont.txt` — ontology descriptions (provided)

## Pipeline: 4 Stages

| Stage | Script | Input | Output |
|-------|--------|-------|--------|
| 1. Candidate Generation | `scripts/pare_pred_candidates.py` | `eng_Latn_final.jsonl` | `nyt_candidates.jsonl` |
| 2. Bag Selection | `scripts/ret_give_bag_eng.py` | `nyt_candidates.jsonl` | `sent_give_5_bag_*.json` |
| 3. Sentence Selection | `scripts/ret_single_from_bag_eng.py` | `sent_give_5_bag_*.json` | `*_single_selected.json` |
| 4. LLM Inference | `scripts/infer_api.py` | `*_single_selected.json` | LLM predictions |

## Running the Main Pipeline (NYT10m)

```bash
# 0. Download benchmark data
uv run python scripts/download_nyt10m.py --make_hydre_aliases

# 1. Stage 1: PARE candidate generation
uv run python scripts/pare_pred_candidates.py \
    --query_jsonl datasets/nyt10m/eng_Latn_final.jsonl \
    --out_jsonl datasets/nyt10m/nyt_candidates.jsonl \
    --rel2id_file datasets/nyt10m/nyt10m_rel2id.json \
    --ckpt PARE/ckpt/bert-base-uncased_512_32_5_1e-5_772_nyt10m_sep_na.pth.tar \
    --device cuda:0 --batch_size 64

# 2. Stage 2: Bag selection (BERTScore + PARE)
uv run python scripts/ret_give_bag_eng.py \
    --train_path datasets/nyt10m/nyt10m_train_opennre_clean_hfmre256.jsonl \
    --query_path datasets/nyt10m/eng_Latn_final.jsonl \
    --out_dir datasets/nyt10m \
    --pare_top_file datasets/nyt10m/nyt_candidates.jsonl \
    --rel2id_path datasets/nyt10m/nyt10m_rel2id.json \
    --no-mask --num_workers 32 --retrieval_batch_size 200

# 3. Stage 3: Sentence selection
uv run python scripts/ret_single_from_bag_eng.py \
    --bag_file datasets/nyt10m/sent_give_5_bag_pare5_e5-large-v2_nonna_en_distinct_mskFalse_MAX_sent_nottune.json \
    --out_dir datasets/nyt10m/ \
    --rel2id_path datasets/nyt10m/nyt10m_rel2id.json \
    --pare_ckpt PARE/ckpt/bert-base-uncased_512_32_5_1e-5_772_nyt10m_sep_na.pth.tar

# 4. Stage 4: LLM inference (set OPENAI_API_KEY or TOGETHER_API_KEY in .env)
uv run python scripts/infer_api.py \
    --example_path datasets/nyt10m/sent_give_5_bag_pare5_e5-large-v2_nonna_en_distinct_mskFalse_MAX_sent_nottune_single_selected.json \
    --source_path datasets/nyt10m/eng_Latn_final.jsonl \
    --model gpt-4-turbo
```

## Ablations

4 ablation scripts in `ablations/` that replace components of the pipeline with baselines. Each runs through Stages 1-3 and produces a `*_single_selected.json` ready for Stage 4.

### Ablation 1: Without Semantic Similarity
Removes BERTScore/sentence embeddings. Retrieval uses PARE scores only.
```bash
uv run python ablations/ret_give_bag_not_sent_sim.py \
    --train_path datasets/nyt10m/nyt10m_train_500lines.jsonl \
    --query_path datasets/nyt10m/selected_20_queries.jsonl \
    --out_dir datasets/nyt10m \
    --pare_top_file datasets/nyt10m/nyt_candidates_20.jsonl \
    --rel2id_path datasets/nyt10m/nyt10m_rel2id.json \
    --no-mask --num_workers 32 --retrieval_batch_size 200

uv run python scripts/ret_single_from_bag_eng.py \
    --bag_file datasets/nyt10m/not_sent_sim_5_bag_pare5_e5-large-v2_nonna_en_distinct_mskFalse_MAX_sent_nottune.json \
    --out_dir datasets/nyt10m/ \
    --rel2id_path datasets/nyt10m/nyt10m_rel2id.json \
    --pare_ckpt PARE/ckpt/bert-base-uncased_512_32_5_1e-5_772_nyt10m_sep_na.pth.tar
```

## Notes

- Stage 1 requires a PARE checkpoint matching your dataset (NYT: 25 relations, Wiki: 80 relations).
- Stage 4 requires `OPENAI_API_KEY` or `TOGETHER_API_KEY` in a `.env` file.
- Stage 4 requires API keys — set `OPENAI_API_KEY` or `TOGETHER_API_KEY` in `.env`.
